# Zaphkiel-bot
一个支持微信ClawBot方式交流的简易的图形化 AI 沟通界面
一款基于 Tkinter 开发的图形化聊天客户端，支持对接本地 Ollama 服务和外部豆包 API，集成微信消息监听功能，可灵活配置上下文轮数、系统提示词等参数。
功能特性
核心功能
🎯 双 API 模式支持：无缝切换本地 Ollama 模式和外部豆包 API 模式
💬 完整聊天交互：支持文本输入、流式回复展示、思考过程实时显示
⚙️ 灵活的上下文配置：可自定义保留的对话轮数，支持系统提示词永久保留
📜 聊天记录管理：自动保存 / 加载聊天历史，支持导出为文本文件
🔧 丰富的自定义设置：可配置模型名称、API 地址、角色名称、系统提示词等
📱 微信模式：支持监听微信消息并自动回复（需配套 auth.py 和 api.py）
UI 特性
简洁易用的图形化界面，支持中文显示
状态栏实时显示当前状态（准备就绪 / 思考中 / 微信模式）
快捷键支持（Ctrl+N 新建对话，Ctrl+Q 退出）
消息区分不同角色颜色，提升可读性
环境要求
基础依赖
Python 3.8+
必要的 Python 库：
bash
运行
pip install tkinter requests pillow openai
注：tkinter 通常随 Python 安装，若缺失可通过 apt-get install python3-tk（Linux）或下载对应安装包（Windows/Mac）安装
可选依赖
Ollama 模式：需本地部署 Ollama 服务（默认端口 11434）
微信模式：需配套 auth.py（微信登录认证）和 api.py（微信消息收发）
豆包 API 模式：需有效的豆包 API 密钥和基础地址
快速开始
1. 安装依赖
bash
运行
pip install requests pillow openai
2. 运行程序
bash
运行
python ollama-engine.py
3. 基础配置
Ollama 模式（默认）：确保本地 Ollama 服务已启动（ollama serve），默认模型为 deepseek-r1:14b
豆包 API 模式：
点击「设置」→「切换 API 模式」选择外部模式
配置 API 密钥和基础地址（「设置」→「设置 API 地址」）
配置豆包模型名称
界面说明
主界面布局
菜单栏：包含文件、设置、历史、帮助四大功能菜单
上下文配置区：可设置保留的对话轮数（0 = 不传递上下文，正数 = 保留最近 N 轮）
聊天历史区：显示所有聊天记录，不同角色消息以不同颜色区分
输入区：支持多行文本输入，回车发送（Shift + 回车换行）
状态栏：显示当前程序状态（准备就绪 / 思考中 / 微信模式等）
核心操作
发送消息
在输入框中输入文本，点击「发送」按钮或按回车键发送
发送后程序会自动调用对应 API 并实时展示回复
思考过程会以灰色斜体显示，回复内容正常展示
配置上下文
在「上下文配置」区域输入需要保留的轮数
点击「保存配置」按钮生效
配置会自动保存到 chat_config.json
切换 API 模式
点击「设置」→「切换 API 模式」，在 Ollama 和外部 API 间切换
切换后需确保对应 API 配置正确
微信模式
点击「设置」→「开启微信模式」，程序会自动调用 auth.py 进行微信登录认证
认证成功后会自动监听微信消息，收到消息后会：
在聊天界面显示微信用户消息
自动填充到输入框并发送
将 AI 回复发送给对应的微信用户
点击「设置」→「关闭微信模式」可停止监听
配置文件说明
程序会自动生成以下配置文件：
chat_history.json：保存聊天历史记录（包含消息内容、角色、时间戳）
chat_config.json：保存用户配置（上下文轮数、API 模式、模型名称等）
data/credentials.json（微信模式）：保存微信登录凭证
菜单功能详解
文件菜单
新建对话：清空当前聊天记录，重置上下文
保存聊天记录：将当前聊天记录保存到 chat_history.json
导出为文本：将聊天记录导出为纯文本文件
退出：关闭程序
设置菜单
设置模型：修改当前使用的模型名称
切换 API 模式：在 Ollama 和外部 API 间切换
设置 API 地址：修改 Ollama / 豆包 API 的基础地址
开启 / 关闭微信模式：控制微信消息监听功能
设置系统提示词：修改 AI 的系统提示词（人设）
设置角色名称：自定义用户 / 助手 / 系统的显示名称
历史菜单
查看所有聊天记录：显示完整的聊天历史
清除当前聊天记录：清空当前界面的聊天记录
帮助菜单
查看当前上下文：显示当前传递给 API 的上下文内容
关于：显示程序版本信息
使用说明：显示简易使用指南
常见问题
Q1: 提示 "正在生成回复，请稍候！" 但无响应
检查 API 服务是否正常运行（Ollama / 豆包 API）
查看状态栏是否有错误提示
确认网络连接正常（豆包 API 模式）
Q2: 微信模式无法开启
确保 auth.py 和 api.py 存在于同一目录
检查微信登录凭证是否正确生成（data/credentials.json）
确认微信接口配置（token、bot_id、base_url）完整
Q3: 上下文配置不生效
确认已点击「保存配置」按钮
检查 chat_config.json 是否有写入权限
重启程序后再次尝试
Q4: 豆包 API 调用失败
检查 API 密钥是否正确
确认基础地址和模型名称配置正确
检查网络是否能访问豆包 API 地址
